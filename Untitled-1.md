## Шаг № 1: Базовая структура проекта и верстка каркаса

### 1. Техническое задание на этап
Создай базовую структуру веб-приложения «Личный планировщик задач на неделю». 
Стек: HTML5, CSS3, JavaScript (без фреймворков).

Задачи:
1. Создай файл `index.html`. Добавь базовую разметку:
   - Шапка: заголовок приложения, место под текст прогресса, пустой прогресс-бар, кнопка «Сбросить все задачи».
   - Недельная доска: горизонтальный контейнер с 7 колонками (от понедельника до воскресенья). В каждой колонке: заголовок дня, счетчик задач (пока 0), контейнер для карточек, кнопка «Добавить задачу».
   - Скрытое модальное окно для подтверждения сброса.
   - Контейнер для всплывающих уведомлений (toast).
   - Подключи `styles.css` и `app.js` (с атрибутом defer).
2. Создай файл `styles.css`. Напиши стили:
   - Светлая тема, минимализм, нейтральный фон (белый или светло-серый).
   - Горизонтальная раскладка колонок (flex или grid). При ширине экрана менее 1280px (и до 1024px) допустима горизонтальная прокрутка контейнера.
   - Базовые стили для карточек задач, кнопок и инпутов.
3. Создай файл `app.js`. Напиши базовую инициализацию:
   - Массив с идентификаторами дней недели (`monday`, `tuesday`...).
   - Функция `init()`, которая запускается при загрузке DOM, находит основные элементы и выводит в консоль "App initialized".

### 2. Инструкция для самопроверки (Агенту)
- Убедись, что все 3 файла созданы и связаны друг с другом.
- Проверь, что вёрстка не использует внешние UI-библиотеки (Tailwind, Bootstrap и т.д.).
- Проверь консоль на отсутствие ошибок загрузки скриптов или стилей.

### 3. Инструкция для ручной проверки (Пользователю)
* **Что сделать:** Открой файл `index.html` в браузере (двойным кликом или через Live Server).
* **На что обратить внимание:** Наличие шапки и 7 ровных колонок по дням недели. Горизонтальный скролл появляется только при сужении окна. Открой консоль разработчика (F12).
* **Ожидаемый результат:** Интерфейс отображается корректно. В консоли есть сообщение "App initialized", нет красных ошибок.
